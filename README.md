# Dinner Wheel 🎡

A tiny web app for Mark + Jenny: dump meal ideas into a shared bank, pick how many
dinners to plan this week (2, 3 or 4), then spin the wheel to decide. The spin
**locks in** the meal — each person gets one veto per week, that's the game.
A running tally of meals cooked this month and a combined shopping list so the
week is visible before the shop.

**Live app:** https://marktran4.github.io/dinner-wheel/

## Put it on your phone

1. Open the link above in Safari.
2. Share button → **Add to Home Screen**.
3. Open it from the home screen icon — it runs full-screen like a normal app.

## How to play

- **Ideas tab** — add meal ideas. Optionally list ingredients so the
  shop list builds itself. Tap an idea to edit or delete it.
- **This week tab** — pick 2/3/4 dinner slots. Fill a slot by spinning or picking.
  Tap the day chip to assign a day. Hit **Cooked it** when it's done — that bumps
  the monthly tally at the top.
- **Wheel tab** — spin to fill the next empty slot. The result locks in unless
  you spend your one weekly veto. Meals cooked in the last 2 weeks sit out
  unless you turn on "include recent".
- The tally at the top counts every meal cooked this month and resets on the 1st.
  Weeks roll over automatically every Monday.

## Turning on shared sync (one-time, ~10 minutes)

Out of the box each phone keeps its own data ("single-phone mode" banner shows).
To sync between phones it uses Firebase's free tier:

1. Go to https://console.firebase.google.com → **Add project** → name it
   `dinner-wheel`, turn off Analytics, create.
2. In the project: **Build → Realtime Database → Create database**. Pick
   Singapore (`asia-southeast1`). Start in **locked mode**.
3. In the database **Rules** tab, replace the rules with this and publish:

   ```json
   {
     "rules": {
       "mealwheel": {
         "$house": {
           ".read": true,
           ".write": true
         }
       }
     }
   }
   ```

4. Project settings (gear icon) → **Your apps** → add a **Web app** (`</>`),
   name it anything, skip hosting. Copy the `firebaseConfig` object it shows.
5. Paste that object into `firebase-config.js` in this repo (replace `null`),
   and add the `databaseURL` field from the Realtime Database page if it isn't
   already in the config. (Or just paste the config to Claude and it'll do it.)
6. Open the app with a kitchen code in the link, e.g.
   `https://marktran4.github.io/dinner-wheel/#k=YOUR-RANDOM-CODE`
   (any random string, 8+ letters/numbers/dashes). Send Jenny the same link.
   The code keeps your data on its own private path — anyone without the link
   can't find it.

Notes:
- The Firebase web config is public by design; committing it here is fine.
- Data is meal names only. The kitchen code stays out of the repo (it lives in
  the link/on the phones).
- If phones were used in single-phone mode first: the first phone to connect
  seeds the shared database with everything it has; phones joining later keep
  the shared data plus their own meal ideas (idea banks are merged). The monthly
  tally follows the shared data when a later phone joins.

## Dev notes

- One static page: `index.html` (all CSS/JS inline), `sw.js` offline shell,
  `manifest.webmanifest` + icons for the home-screen install.
- No build step, no framework, no server. Firebase Realtime Database (compat
  CDN) is loaded only when `firebase-config.js` has a config.
- Icons are generated by `../make_icons.py` (pure stdlib).
- Source of truth lives in the `ea-automation` repo at `projects/meal-wheel/`;
  this repo is the deploy target for GitHub Pages.
